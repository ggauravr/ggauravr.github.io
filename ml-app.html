The app(my ML project, mentored by Prof. Jihun Hamm) uses sensor data(activity recognition API) from Android phones, along with the day/week information, to try to classify if the user is in class or not(binary classifier). To test the online learning on mobile devices, I need help with collecting data and learn the model and so I request everyone using an Android phone to install and use the app for at least a week, so I could get enough samples to train on. Thanks.

Instructions for installation:

  1. Gingerbread
    Settings -> Applications -> Unknown Sources(check this), to allow installing the apk, not from App Store
  2. Android 4+
    Settings -> Security -> Unknown Sources(check this)

Details of the app:

  1. Calendar Screen : has grids for 7 days a week and half an hour time blocks. The time indicated on the left, shows that starting time of the block. Mark/click the blocks where you have class. Round off to the nearest half hour block. e.g : if you've class starting at 02:20 PM, mark the block starting at 14:30, and if ending at 04:55 PM, mark the block starting at 16:30.
    Options in the screen :
    • - Ignore - ignores the changes done in the calendar, goes to last saved state
    • - Clear - clears the entire calendar
    • - Save - saves the changes
    • - Done - save and navigate to the data collection screen
    Please enter the class schedule when you have classes, and exam schedule when you've exams, which probably will be starting next Tuesday.
  2. Data Collection Screen : has button to toggle the data collection service. Once started, the service keeps running in the background, even after the application is closed.
    Options in the screen :
    - Reschedule - back to calendar, to change the schedule
  3. Data Sync/Network Communication : after every 5 samples created, the app checks for network connectivity, and if present, sync the classifier model and the sample over the network. Size of the data communicated is negligible, in the order of some bytes(not even KB).
  4. Permissions for the app : The app requires permissions to detect your activity(running, still, moving so on), to connect to the network, and to run after reboot.
